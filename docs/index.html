Dashboard Builder v2.0.0

Professional Real-Time Telemetry Visualization Platform for monitoring and analyzing sensor data, system metrics, and telemetry streams.

Developed by Team Ignition Software Department at Vellore Institute of Technology, Chennai

Dashboard Builder is a professional-grade telemetry visualization platform designed for real-time monitoring of sensor data from rockets, drones, embedded systems, and other data-acquisition applications.

Features

Real-time Visualization

Stream and visualize sensor data in real-time from multiple sources including Serial, TCP, UDP, and built-in simulators.

Multiple Widget Types

7+ widget types including graphs, gauges, tables, LED indicators, and GPS maps for comprehensive data monitoring.

Data Logging

Comprehensive data logging to CSV and JSON formats with configurable buffer sizes and parameter selection.

Professional UI

Industry-ready dark theme interface with customizable layouts, dockable widgets, and multi-tab support.

System Requirements

Minimum Requirements

  • OS: Windows 10/11, Linux (Ubuntu 20.04+), macOS 10.15+
  • Python: 3.8 or higher
  • RAM: 4 GB
  • Display: 1280x720 resolution

Recommended Requirements

  • OS: Windows 11, Linux (Ubuntu 22.04+), macOS 12+
  • Python: 3.10 or higher
  • RAM: 8 GB or more
  • Display: 1920x1080 or higher resolution
  • Network: Ethernet for TCP/UDP sources

Installation

Step 1: Install Python

Ensure Python 3.8+ is installed on your system:

python --version

Step 2: Clone Repository

git clone https://github.com/teamignitionvitc/Dashboard-Builder.git
cd Dashboard-Builder

Step 3: Install Dependencies

pip install -r requirements.txt

Required packages: PySide6, pyqtgraph, numpy, pyserial

Step 4: Verify Installation

python main.py

Quick Start Guide

1. Launch Application

python main.py

2. Create New Dashboard

  1. Click "Create New Dashboard" on the welcome screen
  2. Enter the 4-phase dashboard creation wizard

3. Configure Connection

For Testing (No Hardware):

  • Select Connection Mode: dummy
  • Click "Start Dashboard"

For Serial Hardware:

  • Select Connection Mode: serial
  • Choose Serial Port from dropdown
  • Set Baudrate (e.g., 115200)
  • Configure Data Format (e.g., json_array)
  • Set Channel Count

4. Configure Parameters

  1. Go to Edit → Manage Parameters
  2. Click "Add..." to create new parameter
  3. Fill in parameter details (ID, Name, Array Index, Unit, Thresholds)
  4. Click OK to save

5. Add Widgets

  1. Click "Add Widget..." button
  2. Select widget type
  3. Choose parameters to display
  4. Configure widget options
  5. Click OK

Dashboard Interface

Top Bar (Header)

  • Dashboard Title: Customizable title (right-click to edit)
  • Connection Status: Real-time connection indicator
  • Stream Status: STREAMING, PAUSED, or AWAITING PARAMETERS
  • Logging Status: Shows if data logging is active
  • Pause Button: Pause/resume data stream

Status Bar (Bottom)

  • System Clock: Current date and time
  • Connection Details: Active connection information
  • Uptime: Dashboard runtime
  • RX: Total bytes received
  • Packets: Total packet count
  • Rate: Packets per second

Tip: Right-click on any widget for quick actions like rename, float/dock, or close.

Managing Parameters

Parameter Properties

Property Description Example
ID Unique identifier (no spaces) accel_x
Name Display name Acceleration X
Array Index Position in data packet 3
Unit Measurement unit m/s²
Thresholds Alarm levels Low: -20, High: 20

Adding Parameters

  1. Navigate to Edit → Manage Parameters → Add...
  2. Fill in all required fields
  3. Set threshold values in ascending order
  4. Click OK

Important: Ensure Array Index matches the position of your data in the incoming packet.

Data Logging

Configuring Data Logging

  1. Go to Data Logging → Configure Logging...
  2. Select format (CSV or JSON)
  3. Choose output file path
  4. Select parameters to log
  5. Set buffer size (10-1000)

Buffer Size Recommendations

Data Rate Buffer Size Write Frequency
1-10 Hz 100 Every 10-100 seconds
10-50 Hz 200-500 Every 4-50 seconds
50+ Hz 500-1000 Every 10-20 seconds

Note: Larger buffers reduce disk I/O but increase memory usage and potential data loss on crash.

Troubleshooting: Connection Issues

Serial Port Not Found

Solutions:

  • Check device is connected
  • Install device drivers
  • Check port permissions (Linux: sudo usermod -a -G dialout $USER)
  • Click "Refresh" button to update port list

Network Connection Issues

Solutions:

  • Verify IP address and port
  • Check firewall settings
  • Ping host to verify connectivity
  • For UDP: Ensure device is sending to correct port
  • For TCP: Verify server is listening

Critical: Always stop the data stream before changing connection settings to avoid errors.
